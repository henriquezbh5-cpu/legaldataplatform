YAML rule engine Three lines of defense - line 2

Each rule is declarative (not_null, regex, in_set, row_count_range...) and runs against the full Polars DataFrame. Errors quarantine the row; warnings emit Prometheus counters.

Defense layers 3 of 3

Line 1
Pydantic v2 (structural)
Type, format, required-fields. Rust-backed core. Runs on every row at ingest.
Line 2
YAML rule engine (business)
Domain rules: SHA-256 format, taxonomy, jurisdiction whitelist, row-count range.
Line 3
Great Expectations (statistical)
Distribution checks, regression detection, data-doc generation.
