02 · Arquitectura

Medallion + Source of Truth

Bronze (raw inmutable) → Silver (clean) → Gold (business-ready). PostgreSQL como fuente de verdad operativa; S3 como archivo histórico reprocesable. 

┌─────────────────────────────────────────────────────────────────┐
│  FUENTES: REST APIs · CSV/SFTP · DBs externas · S3 Parquet      │
└─────────────────────────────┬───────────────────────────────────┘
                              ↓
              EXTRACTORS (async Python, lineage-tagged)
                              ↓
              BRONZE en S3 (Parquet Snappy, inmutable)
                              ↓
  ┌──────────────────────────────────────────────────────────────┐
  │ VALIDATION: Pydantic → YAML rule engine → Great Expectations │
  └─────────────────────────────┬────────────────────────────────┘
                                ↓
                  ┌─────────────┴─────────────┐
                  ↓                           ↓
          QUARANTINE (S3)              TRANSFORMERS
          rechazos con razón           normalize · SCD2 · enrich
                                            ↓
                                 SILVER en S3 + PostgreSQL
                                            ↓
                        LOADERS: COPY BULK + UPSERT staging
                                            ↓
             POSTGRESQL (SoT) · particionado · índices · MViews
                                            ↓
                                 GOLD agregados (refresh CONCURRENTLY)
                                            ↓
                          BI · APIs · ML · Reportes

Orquestación: PREFECT · Observabilidad: structlog+Prometheus+Grafana
Infra: TERRAFORM · Docker Compose · AWS (S3/Lambda/SQS/Glue/Aurora)

06 · Quickstart

Cómo correrlo en tu máquina

Scripts de bootstrap cross-platform. Prerequisitos: Python 3.11+, Docker, Git.

Windows PowerShell

git clone https://github.com/TU_USER/legaldataplatform
cd legaldataplatform
powershell -ExecutionPolicy Bypass -File scripts\setup.ps1

macOS / Linux / WSL

git clone https://github.com/TU_USER/legaldataplatform
cd legaldataplatform
bash scripts/setup.sh

El script automatiza: virtualenv, dependencias, Docker, migraciones y seed data. Luego make pipeline ejecuta el flow completo en menos de 1 minuto.
